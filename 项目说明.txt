智能组卷系统 V3 - 项目文件说明文档

本文档详细说明了“智能组卷系统 V3”项目中各个目录及文件的作用。本项目采用前后端分离架构，前端使用 Vue 3 + Vite + Element Plus，后端使用 Python Flask + SQLAlchemy + MySQL。

目录结构概览
------------
- /backend            # 后端项目目录（Flask）
- /frontend           # 前端项目目录（Vue3）
- README.md           # 项目整体启动与安装说明
- 项目功能文档.txt    # 详细的功能需求说明
- 项目数据库设计.txt  # 数据库表结构设计说明

1. 后端部分 (backend/)
核心文件：
- run.py                # 后端服务的启动入口脚本。运行此文件可启动 Flask 开发服务器。
- requirements.txt      # Python 依赖包列表（包含 Flask, SQLAlchemy, PyMySQL, requests 等）。
- .env.example          # 环境变量配置示例文件。用于配置数据库连接字符串、DeepSeek API Key 等敏感信息。

应用源码 (backend/app/):
- __init__.py           # Flask 应用工厂函数。负责初始化 app、注册数据库扩展、注册 API 蓝图。
- config.py             # 应用配置类。定义数据库 URI、密钥等配置项。
- db.py                 # 数据库模块。包含 SQLAlchemy 实例定义、数据库初始化与会话管理辅助函数。

API 接口模块 (backend/app/api/):
- __init__.py           # API 蓝图注册入口。统一管理各个业务模块的路由前缀。
- dicts.py              # 基础字典接口。提供科目、题型、难度等基础数据的查询接口。
- textbooks.py          # 教材管理接口。实现教材增删改查、章节树结构的维护、章节 Excel 导入及概要生成。
- questions.py          # 题库管理接口。实现题目 CRUD、多条件检索、Excel/Word 批量导入功能。
- ai.py                 # AI 模块接口。处理 AI 智能出题请求、题目入库（待校验状态）及人工校验流程（通过/拒绝）。
- papers.py             # 试卷管理接口。实现智能组卷算法、试卷保存、试卷编辑（分值/顺序）、Word 导出及版本管理。
- answer_sheets.py      # 答题卡接口。管理答题区域样式库、答题卡生成、题目与样式的绑定关系。

业务服务 (backend/app/services/):
- deepseek.py           # DeepSeek AI 服务封装。负责构造提示词并调用 DeepSeek API 进行题目生成与内容分析。


2. 前端部分 (frontend/)
核心配置：
- package.json          # Node.js 项目配置文件。定义项目依赖（Vue, Element Plus 等）及 npm 运行脚本。
- vite.config.js        # Vite 构建工具配置。配置服务器端口、别名及构建选项。
- index.html            # 单页应用（SPA）的 HTML 入口文件。

源码目录 (frontend/src/):
- main.js               # Vue 应用入口文件。挂载 Vue 实例，引入全局样式与插件。
- App.vue               # 根组件。定义了系统的整体布局（Header, Sidebar, Main）及导航菜单。
- style.css             # 全局样式文件。

路由与网络 (frontend/src/):
- router/index.js       # 路由配置。定义 URL 路径与页面组件的映射关系。
- api/http.js           # HTTP 请求封装。基于 Axios 封装，统一处理 API Base URL 和响应拦截。

页面视图 (frontend/src/views/):
- DictsPage.vue         # [基础字典] 页面。展示科目列表、题型列表等基础数据。
- TextbooksPage.vue     # [教材管理] 页面。提供教材维护、章节树编辑及章节概要生成功能。
- QuestionsPage.vue     # [题库管理] 页面。提供题目的查询、录入、编辑及批量导入入口。
- AiReviewPage.vue      # [AI 校验] 页面。用于发起 AI 出题任务，并对 AI 生成的待校验题目进行人工审核。
- PaperGeneratePage.vue # [智能组卷] 页面。提供组卷策略（JSON）编辑器，支持在线预览策略并生成试卷。
- PapersManagePage.vue  # [试卷管理] 页面。查看已生成的试卷，支持二次编辑、Word 导出及历史版本下载。
- AnswerSheetsPage.vue  # [答题卡] 页面。管理答题卡样式库，配置答题卡结构及生成预览。

需要注意的是其中backend/app/config.py中的数据库连接字符串需要根据实际部署环境进行配置，确保能够正确连接到 MySQL 数据库。
此外，DeepSeek API Key 如下sk-3123383874c042e8a16e8d3e93c80810，在config.py的十六行。
请注意推送之前在config.py中删除该密钥以防泄露。
数据库采用远程数据库，不需要在自己电脑上配置数据库环境，数据库名字是exam_system-v3。
项目启动步骤：
1. 克隆代码库到本地。
2. 安装后端依赖：在 backend/ 目录下运行 pip install -r requirements.txt。
3. 配置环境变量：复制 .env.example 为 .env 并填写数据库连接信息和 DeepSeek API Key。打开终端，进入 backend/ 目录，运行命令：
   cp .env.example .env
   然后编辑 .env 文件，填写正确的数据库连接字符串和 DeepSeek API Key。之后保存退出。
4. 启动后端服务：在 backend/ 目录下运行 python run.py。
5. 安装前端依赖：在 frontend/ 目录下运行 npm install。
6. 启动前端服务：在 frontend/ 目录下运行 npm run dev。
7. 访问前端应用：打开浏览器，访问 http://localhost:5173 （默认端口）。
